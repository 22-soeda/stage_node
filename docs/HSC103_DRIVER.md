# HSC-103 ステージドライバ（`Hsc103Stage`）

SIGMA KOKI 製 **HSC-103** コントローラ向けの Python 実装です。実装ファイルは `drivers/hsc103_stage.py` にあり、`AbstractStage`（座標は **mm**）に従います。

## 概要

- **通信**: `pyserial` によるシリアル（既定ボーレート 38400、8N1、**RTS/CTS フロー制御あり**）。
- **プロトコル**: サンプル `HSC-103.py` と同系のテキストコマンド（`Q:` / `A:` / `M:`）。
- **アプリ側の軸対応**: X→コントローラ軸1、Y→軸2、Z（高さ）→軸3。
- **内部表現**: コントローラはステップ単位で指令・応答するため、**steps/mm** で mm と相互変換します。

## `AbstractStage` で定義される API

| メソッド | 動作 |
|---------|------|
| `connect()` | シリアルオープン後、`Q:` で現在ステップを読み、**接続時点を論理原点**（0,0,0 mm）の基準ステップとして保存 |
| `disconnect()` | 移動完了待ちのうえクローズ |
| `move_abs(x,y,z)` | `A:tx,ty,tz` で絶対ステップ移動（論理 mm から目標ステップを計算） |
| `move_rel(dx,dy,dz)` | 現在ステップを `Q:` で取得し、`M:rx,ry,rz` で相対移動 |
| `get_position()` | `Q:` のステップから論理 mm を算出 |
| `set_origin()` | `R:1,1,1` で**コントローラ内部の論理原点を 0** に設定し、`Q:` で 0,0,0 を確認 |
| `is_moving()` / `wait_for_move()` | 移動スレッドの状態・完了待ち |

未接続時は `StageError`、接続失敗時は `StageConnectionError` が送出されます。

## シリアルパラメータ（接続時）

- `bytesize=EIGHTBITS`, `parity=NONE`, `stopbits=ONE`
- `timeout=5`（秒）
- **`rtscts=True`**（ハードウェアフロー制御）

## steps/mm（スケール）

コントローラのステップ解像度は機種・設定に依存します。コード上の既定は **100000 steps/mm**（全軸）。旧既定の 1000 steps/mm では、実機で指令に対し移動量が約 1/100 になる事例があったため、デフォルトを引き上げています。

環境変数で上書きできます。

| 変数 | 意味 |
|------|------|
| `MEASUREMENT_HSC103_STEPS_PER_MM` | 全軸共通の基準値（未設定時 100000） |
| `MEASUREMENT_HSC103_STEPS_PER_MM_X` / `_Y` / `_Z` | 軸別（未設定時は上記基準を使用） |

コンストラクタ引数 `steps_per_mm=(sx, sy, sz)` を渡すと、環境変数より優先されます。

## 送受信コマンド（要点）

- **位置問い合わせ**: `Q:\r\n` → 応答行から正規表現で3つの整数を抽出（軸1〜3のステップ）。
- **絶対移動**: `A:step1,step2,step3\r\n`（負数は `-123` 形式。`_fmt_step` で符号を整形）
- **相対移動**: `M:dr1,dr2,dr3\r\n`

移動コマンド送信後、実装は `readline()` で1行読み捨て（応答消費）し、その後バックグラウンドで完了を監視します。

## 移動の非同期モデル

`move_abs` / `move_rel` はコマンド送出後すぐに戻ります。完了は別スレッドが `Q:` をポーリングし、目標ステップとの差が許容範囲内になるまで待ちます。

- **許容誤差**: コンストラクタ `move_tolerance_steps`（既定 3 ステップ）。いずれか軸がこれを超えると「未到達」とみなしてループ継続。
- **`wait_for_move()`**: 移動スレッドの `join()` でブロック。次の移動や `set_origin` の前にも内部で呼ばれ、連続指令の衝突を防ぎます。

## タイムアウトと「打ち止め」検出

可動域外やリミットで物理位置が目標ステップに届かない場合、`is_moving` が永遠に True のままになるのを避けるため、次のロジックがあります。

- **タイムアウト**: `move_timeout_s`（未指定時は環境変数 `MEASUREMENT_HSC103_MOVE_TIMEOUT_S`、既定 300 秒）。
- **打ち止め**: 開始から `MEASUREMENT_HSC103_MOVE_GRACE_S`（既定 0.5 秒）経過後、誤差が `MEASUREMENT_HSC103_MOVE_STALL_S`（既定 2.0 秒）改善しなければ移動完了とみなしてスレッド終了（ログに警告）。

ポーリング間隔は `poll_interval_s`（既定 0.05 秒）。

## アプリからの選択方法

`StageController` の `connect` で `driver` に `hsc103`（ハイフン可）を指定し、`port` に COM ポート名を渡します。

例（概念）: `connect` アクションで `driver: "hsc103"`, `port: "COM5"`。

ターミナルヘルプでは `connect hsc103 COM5` のような形式が案内されています。

## 依存関係

- `pyserial`（`serial` モジュール）
- プロジェクト共通: `AbstractStage`, `StageError`, `StageConnectionError`

## まとめ

`Hsc103Stage` は HSC-103 の **シリアルテキストプロトコル**をラップし、**mm 座標**と **ステップ指令**を **steps/mm** で橋渡しします。移動は **非同期ポーリング**で完了を追い、**可動域外**では **打ち止め検出**でデッドロックを避けます。実機に合わせて **steps/mm を環境変数またはコンストラクタで調整**することが重要です。
